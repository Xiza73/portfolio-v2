# i18n — Estrategia de Internacionalización

---

## Alcance

Soporte bilingüe **español (ES)** e **inglés (EN)**. Español es el idioma por defecto.

---

## Enfoque: Astro i18n Nativo + react-i18next para Islands

Astro tiene soporte nativo para routing por idioma. Los React islands usan `react-i18next` para acceder a traducciones en el cliente.

### Routing

```
/          → Español (idioma por defecto, sin prefijo)
/en/       → Inglés
```

### Estructura de archivos

```
src/
├── i18n/
│   ├── config.ts          # Idiomas soportados, idioma por defecto
│   ├── utils.ts           # getTranslation(), getLocaleFromUrl()
│   └── messages/
│       ├── es.json        # Traducciones en español
│       └── en.json        # Traducciones en inglés
│
├── pages/
│   ├── index.astro        # Español (default)
│   └── en/
│       └── index.astro    # Inglés
```

## Configuración

### `i18n/config.ts`

```typescript
export const LANGUAGES = {
  es: 'Español',
  en: 'English',
} as const;

export type Language = keyof typeof LANGUAGES;

export const DEFAULT_LANGUAGE: Language = 'es';

export const LANGUAGE_CODES = Object.keys(LANGUAGES) as Language[];
```

### `i18n/utils.ts`

```typescript
import { DEFAULT_LANGUAGE, type Language } from './config';
import es from './messages/es.json';
import en from './messages/en.json';

const translations = { es, en } as const;

export function getTranslation(lang: Language) {
  return translations[lang] ?? translations[DEFAULT_LANGUAGE];
}

export function getLocaleFromUrl(url: URL): Language {
  const [, lang] = url.pathname.split('/');
  if (lang === 'en') return 'en';
  return DEFAULT_LANGUAGE;
}

export function getLocalizedPath(path: string, lang: Language): string {
  if (lang === DEFAULT_LANGUAGE) return path;
  return `/${lang}${path}`;
}
```

## Uso en Componentes Astro

```astro
---
import { getLocaleFromUrl, getTranslation } from '@/i18n/utils';

const lang = getLocaleFromUrl(Astro.url);
const t = getTranslation(lang);
---

<h1 class="pixel-text text-primary">{t.hero.title_line1}</h1>
<p class="pixel-text text-accent">{t.hero.description}</p>
```

## Uso en React Islands

Para React islands, pasar las traducciones como props desde el componente Astro padre:

```astro
---
import { ContactForm } from '@/components/react/ContactForm';
import { getLocaleFromUrl, getTranslation } from '@/i18n/utils';

const lang = getLocaleFromUrl(Astro.url);
const t = getTranslation(lang);
---

<ContactForm
  client:idle
  translations={t.contact.form}
  validationMessages={t.contact.validation}
/>
```

```tsx
// ContactForm.tsx
interface ContactFormProps {
  translations: {
    name: string;
    email: string;
    message: string;
    submit: string;
    // ...
  };
  validationMessages: {
    required: string;
    invalid_email: string;
    // ...
  };
}

function ContactForm({ translations, validationMessages }: ContactFormProps) {
  // usar translations.name, translations.submit, etc.
}
```

> **Nota:** Este enfoque (pasar traducciones como props) es más simple que montar react-i18next para un par de islands. Si en el futuro los islands crecen mucho, se puede migrar a react-i18next con un provider.

## Selector de Idioma

Un botón/link en el header que alterna entre idiomas:

```astro
---
import { getLocaleFromUrl, getTranslation, getLocalizedPath } from '@/i18n/utils';

const lang = getLocaleFromUrl(Astro.url);
const t = getTranslation(lang);
const switchLang = lang === 'es' ? 'en' : 'es';
const switchPath = getLocalizedPath('/', switchLang);
---

<a href={switchPath} class="pixel-text text-primary hover:text-accent text-sm">
  {t.language.switch}
</a>
```

## Reglas

1. **Nunca hardcodear textos** en componentes. Siempre usar claves de traducción.
2. **Los tags retro** (`HELLO_WORLD.EXE`, `PROJECTS.DIR`, etc.) también se traducen, pero se mantienen en estilo "terminal" en ambos idiomas (son parte del branding, no texto funcional).
3. **Español es el idioma por defecto** — la ruta raíz `/` siempre es español.
4. **Mantener ambos archivos JSON sincronizados** — mismas claves en `es.json` y `en.json`.
5. **No usar librerías de i18n pesadas** si el enfoque de props es suficiente para los pocos islands del portafolio.
